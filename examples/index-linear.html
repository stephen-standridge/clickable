Clickable is an easy-to-use, configurable content navigation module. Clickable supports three basic types of navigation: linear, targetted, and nested navigation.

Description

Basic Implementation

The most basic of the implementations is the linear navigation. Linear navigation is great for slideshows and any sequential content. The basic elements for a linear navigation module are:

  • An interaction wrapper element with the class'js-clickable-interaction' by default. This element should contain all other clickable elements.
  • At least one(but really, at least two) content area elements with the class'js-clickable-content-area' by default. These elements represent the content through which the user is navigating.
  • At least one navigation element. For linear navigations, Clickable will look for '.js-clickable-prev' and '.js-clickable-next'

Notes

  • The Clickable will automatically look for the default class for all basic element
  • Will log a no interaction found warning if it cannot find elements of the wrapper class.
  • Will log a multiple interactions found warning if it finds multiple wrappers.
  • Will log a no content warning if it cannot find elements of the content class within the wrapper element.
  • Will log a no navigation warning if it cannot find at least one element of the previous and next class('.js-clickable-prev' and '.js-clickable-next' by default) within the wrapper element.

Javascript

new Clickable({wrapper: '.js-basic-interaction'});

Description

Navigating

After the Clickable is initialized, it will automatically add the 'active' and 'js-active' classes to the first js-content area in the dom. It will also add an active-index and navigated class to the interaction wrapper where index is the current scene visible. It will bind the js-clickable-next element to increase the index and the js-clickable-prev to decrease the index.

Notes

  • The Clickable will default to 'infinite' navigation. Infinite navigation will make all of the content areas an infinite loop.
  • To override this default, you can pass 'infinite: false' to the constructor.
  • Finite navigation makes content areas have a definite start and end..
  • The active-index class is a great way to make content-area-specific styling on the entire Clickable
  • The navigated class implies that the Clickable has navigated to the first content area
  • The Clickable will automatically consider the first content area active and mark it and the interaction as such.

Description

The Meta Controls

The Clickable supports two meta control elements: Start and Clear. The start element is used to prevent the first content area to be automatically active. The start element's existence will interrupt the regular initialization and make the activating of the first content area dependent on the start element click. Clicking the clear element restarts the interaction to the initial state.

Notes

  • To make a start element, the Clickable will look for 'js-clickable-start' by default.
  • To make a clear element, the Clickable will look for 'js-clickable-clear' by default.
  • Only once the start screen has been clicked the Clickable receives the 'navigated' class. The 'navigated' class on the Clickable is a great way to hide the start screen on click.
  • The clear element will remove the 'navigated' class on click as well as restarting the screen to active-0
