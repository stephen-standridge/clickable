Clickable is an easy-to-use, configurable content navigation module. Clickable supports three basic types of navigation: linear, targetted, and nested navigation.

Targets

Targets elements are a different form of navigation. Targetted navigation works from the order that each element (target and content) appears in the page. The first target correspondingly activates the first content element on click.

Notes

  • To create a target element, the Clickable will look for '.js-clickable-target' by default.
  • Target elements should occur in a 1:1 fashion with Content elements.
  • Targetted navigation can be blocked by a start element.
  • Targetted navigation can be cleared by the clear element.
  • Targetted navigation adds the class '.targetted' to the Clickable element.
  • Targetted navigation and linear navigation can coexist peacefully.
  • The clickable will log a no navigation found error if it cannot find a next element, a prev element or a target element.

Indicators

Indicators are elements that are activated in parallell with content elements. Just as content elements are activated in a linear or targetted fashion, so are indicators. Unlike content elements, Indicator elements receive a 'visited' state once they are no longer active.

Notes

  • The Clickable will look for '.js-clickable-indicator' by default
  • Indicator elements mimic the activation of the content elements meaning when content 0 is activated, indicator 0 is active.
  • Indicators add the class '.active' and '.js-active' when they are active
  • Indicators replace the '.active' and '.js-active' with '.visited' and '.js-visited' once they are no longer active.
  • Indicators can either be active or visited, not both.
  • The clickable will log a no content found error if it cannot find either an indicator element or a content element.

Mixing Elements

Any element except the wrapper element can be mixed with any other element type. This is not always the wisest decision. (Start + Clear does nothing good). Some useful combinations:

  • Indicator element + Target element. Allows the controls to reflect the currently active content area
  • Indicator element + Content element. Allows content areas to have a visited state.
  • Clear element + final Content element. A good way to restart the Clickable interaction

Toggle Option

The toggle constructor option is a best used with targetted navigation. When the Clickable's target option is enabled, the content area and indicator will work like normal on the initial click of a target element and then deactivate on the second click of that same target element.

  • By default toggle is set to false
  • To enable, pass {toggle: true} to the Clickable constructor
  • You can also change the clickable's toggle option after construction by declaring Clickable.toggle = true;
  • Toggling only removes the active-index class; it does not remove or change any other global states.

  </div>

    first
  </div>   <div class="js-clickable-target js-clickable-indicator">
    second
  </div>
</div>

Javascript

new Clickable({wrapper: '.js-toggle-interaction', toggle: true});
