Clickable is an easy-to-use, configurable content navigation module. Clickable supports three basic types of navigation: linear, targetted, and nested navigation.

Nested Navigation with Contexts

A Clickable can have navigational contexts that define which navigational elements are tied to which content and indicator elements. By specifying a navigational context, you can enable nested navigation wherein the user can navigate within an activated content element.

Notes

  • By default, the Clickable will not recognize any contexts.
  • The Clickable will look for the data-context attribute to establish a navigational context.
  • If some elements have a data-context and some elements don't, the Clickable will make a default context to manage the elements without a defined context
  • No content and no Navigation warnings are specific to a context. The warning will specify the context.
  • When navigational contexts are present, the default '.active-index' class will be replaced with a '.active-context-index' class.

Mixing Navigation

Although most cases won't require it, Clickable is made to successfully manage any navigation element combination. It is entirely possible for a Clickable interaction to have prev/next, targets, clear and start in the same interaction. Even with all of these elements, the toggle and infinite settings will work. Nothing special is needed to make this happen or work except defining the elements.

Global States

The Clickable has global states which relate to how and if the Clickable has been navigated. These states are added and removed from the wrapper via a class. The five main states are:

  • Initial.
    • No class is added to the interaction
    • All other states are removed
    • This state occurs if the Clickable has a start element before the start element is clicked.
    • This state also occurs if the interaciton has a clear element and after the clear element has been clicked
  • Navigated.
    • The 'navigated' class is added to the Clickable wrapper.
    • This state occurs any time the Clickable is not in the initial state.
  • Linear.
    • The 'linear' class is added to the Clickable wrapper
    • This state occurs when the most recent navigation was a previous or next element click.
    • This state cannot occur the same time as the targetted state
  • Targetted.
    • The 'targetted' class is added to the Clickable wrapper
    • This state occurs when the most recent navigation was a target element click.
    • This state cannot occur the same time as the linear state
  • Active-index.
    • The 'active-index' class is added to the Clickable wrapper where index is the current content area active.
    • This state occurs any time the Clickable is navigated.
    • This state is removed in the initial state.
    • This state cannot occur the same time as other active-index states.

Clickable Defaults

To make things flow smoothly, Clickable has a few defaults; all of which can be overridden in the constructor. The defaults are as follows:

{
  wrapper: '.js-clickable-interaction',
  content: '.js-clickable-content-area',
  indicators: '.js-clickable-indicator',
  navigation: {
    targets: '.js-clickable-target',
    prev: '.js-clickable-prev',
    next: '.js-clickable-next',
  },
  meta: {
    clear: '.js-clickable-clear',
    start: '.js-clickable-start',
  },
  infinite: true,
  toggle: false
}

Example

Javascript

new Clickable({
  wrapper: '.new-wrapper-class',
  content: '.new-content-class',
  indicators: '.new-indicator-class',
  navigation: {
    targets: '.new-target-class',
    prev: '.new-prev-class',
    next: '.new-next-class',
  },
  meta: {
    clear: '.new-clear-class',
    start: '.new-start-class',
  },
  infinite: false,
  toggle: true
})
